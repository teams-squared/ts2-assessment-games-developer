Instructions for Games Developer Technical Assessment

GoalComplete a technical assessment demonstrating Java / JVM fundamentals and systems thinking through reasoning questions and practical coding tasks.

ScoreBased on correctness, clarity of reasoning, tradeoffs, and code quality.

Maximum Time60 minutes

AllowedJava, Clojure, or clear pseudocode

If any detail is underspecified, state your assumptions and proceed. Correctness matters, but clarity of reasoning and tradeoffs matter more.

Project Details

Description

  • This assessment evaluates your understanding of Java / JVM fundamentals and systems thinking through a combination of reasoning questions and practical coding tasks.
  • You will be asked to reason about memory representation, performance optimization, debugging strategies, data structures, and algorithm design.
  • The assessment is divided into three main sections covering reasoning, data structures, and advanced technical tasks.

Your Role

  • As part of this assessment, you will answer reasoning questions and implement or describe solutions to technical problems.
  • All tasks have been described step-by-step below. Simply follow these steps to complete the assessment.
  • You may use Java, Clojure, or clear pseudocode for coding tasks. Bullet points are fine for reasoning questions where appropriate.

Tech Stack

  • Java / JVM You'll work with Java or Clojure on the JVM, focusing on memory management, object allocation, and performance considerations.
  • Data Structures You'll design and implement data structures for 3D grids, considering time complexity and memory optimization.
  • Algorithms You'll describe algorithms for ray traversal and grid operations.
  • Game Development You'll demonstrate understanding of real-time game loops, asset pipelines, and 3D model integration.
  • Version Control Git and GitHub are used to host this project's repository and for submitting your work.
  • Since this assessment will primarily use Git & GitHub, you will need a GitHub account and some experience working with the Git CLI.

Task 1 - Setting up your development environment

  1. Fork this GitHub repository onto your personal GitHub account and clone it to your machine - this folder contains the structure for the assessment you will be working on. The time allocation (max. 60 minutes) will start from the point you fork this repository.
  2. Navigate to this repo's folder from your machine. Inside this folder you will find the assessment structure. This is where you will spend your time writing answers and code. You can ignore the folder named /instructions.
    • Ensure that you only edit / append code to the respective files within these folders which have been specified in the instructions. Do not change any other files, as this may result in our test scripts failing to score your submission.
    • Your final submission will be made as a pull request to this Github repository. You will receive your score as a comment to this pull request on GitHub itself.
    • Feel free to commit and push regularly to your personal fork of our repository. However, only submit a pull request to our GitHub repository after you complete the entire assessment.
  3. Open the repo's folder using your favorite text editor / IDE.
  4. Set up your development environment with Java or Clojure as needed. Ensure you have a JVM installed and can compile and run code.
  5. The tasks below will specify if any additional files need to be added to this directory or its subdirectories. Don't add any additional files otherwise.
Task 1 is now complete. You may now move on to Task 2.

Task 2 - Reasoning & Systems Thinking

Spend around 20 minutes

Answer concisely. Bullet points are fine where appropriate. Write your answers in the appropriate files as specified below.

  1. Memory representation & tradeoffs

    You need to represent a very large 3D world where most cells are empty.

    • Describe two different ways you could store this data in memory.
    • For each approach, explain:
      • When it works well.
      • When it becomes problematic.

    Do not assume any engine or framework.

    Write your answer for this question in the task2/question1.md Markdown file.

  2. JVM references, allocation, and real-time loops

    In a real-time game loop running on the JVM:

    • Explain how object allocation and references can affect performance.
    • Describe two concrete practices you would use to keep frame times stable.

    Keep the answer practical, not theoretical.

    Write your answer for this question in the task2/question2.md Markdown file.

  3. Performance debugging under ambiguity

    A player reports sudden frame-time spikes when moving the camera.

    Without seeing the code, list the first few things you would check, in order.

    For each, briefly state what you are trying to confirm or rule out.

    Write your answer for this question in the task2/question3.md Markdown file.

Task 2 is now complete. Commit and push your work to your personal forked repository. You may now move on to Task 3.

Task 3 - Data Structures

Spend around 15 minutes

You are working with a fixed-size 3D grid:

  • width = 32
  • height = 32
  • depth = 32

Each cell stores a single integer value.

  1. Propose a data structure to store this grid in memory.
  2. Write (or describe in pseudocode):
    • get(x, y, z)
    • set(x, y, z, value)
  3. State:
    • Time complexity of each operation.
    • One optimization you would apply if most cells are empty.

    4. You may assume all coordinates are zero-based.

Write your solution for this task in the task3.md Markdown file.

Task 3 is now complete. Commit and push your work to your personal forked repository. You may now move on to Task 4.

Task 4 - Ray Traversal + Asset Integration Integration

Spend around 30 minutes

  1. Part A - Grid Ray Traversal

    You are given a 2D grid where:

    • 0 = empty cell
    • 1 = solid cell

    A ray is defined by:

    • Origin (x0, y0) (floating point).
    • Direction (dx, dy) (normalized).

    Task

    Describe an algorithm (code or pseudocode) that returns the first solid cell the ray intersects, or null if none is hit.

    Your answer should include:

    • How the ray advances through the grid.
    • How boundaries are handled.
    • Why this approach is efficient.

    Exact math precision is not required.

    Write your solution for Part A in the task4/part-a.md Markdown file.

  2. Part B - Integrating Blender assets into a custom engine

    You are importing models created in Blender into a custom game engine.

    Answer the following:

    1. When exporting a static 3D model from Blender, list three technical aspects you would verify or adjust before integration.
      • Use correct terminology (e.g., transforms, UVs, normals, etc.).
    2. Explain one issue that commonly occurs during asset import and how you would diagnose it.
    3. Briefly describe how you would structure an asset pipeline so that artists can update models without breaking the game.

    Keep answers concise and practical.

    Write your solution for Part B in the task4/part-b.md Markdown file.
